## 0. 핵심 컨셉 & 목표 UI

**한 줄 정의**: 우쿨렐레 학습자가 코드 운지를 보기 좋게 모아 보는 **레퍼런스 SPA**. 흰 배경 + 부드러운 그림자(뉴모피즘) 톤, 가로 화면(16:9)을 꽉 채우는 레이아웃.

**3단계 화면 흐름 (‘수준’ 명명)**
```
[수준1: 메인 - 코드 종류 선택] ──클릭──▶ [수준2: 갤러리 - 루트음 선택] ──클릭──▶ [수준3: 상세 - 대형 원본 다이어그램]
        ▲                                          │                                       │
        └──────────────── 뒤로 / 홈(로고) 버튼 ◀────┴───────────────────────────────────────┘
```

**목표 UI 동작**
1. 전체 배경은 흰색(또는 아주 밝은 회색), 부드러운 그림자의 **뉴모피즘** 톤.
2. 상단: 왼쪽에 **돋보기 아이콘 + 길고 둥근(pill) 검색창**, 오른쪽에 연핑크 **`Lesson Designer`** 타이틀(글자 간격 넓음).
3. 첫 화면에 코드 종류를 고르는 **원형 버튼 12개**가 2줄로 배치된다 (16:9 가로폭을 활용).
   - 1행: `Major` · `7` · `minor` · `minor7` · `sus4` · `Major 7`
   - 2행: `6` · `7sus4` · `add2` · `m7(b5)` · `diminish` · `Augment`
4. 각 원형 버튼 = **흰 내부 + 색상 외곽선(링) + 부드러운 그림자**로 살짝 떠 있는 느낌. hover/클릭 시 부드럽게 살짝 커진다.
5. `Major`를 클릭하면 Major 코드 카드 그리드로 전환(`C`, `D`, `E`, `F`, `G`, `A`, `B` 등).
6. 각 코드 카드 = **작은 코드 다이어그램(미니)** + 라벨.
7. 특정 코드를 클릭하면 화면 중앙에 코드명이 **크게** 뜨고, **큰 원본 다이어그램**이 화면을 가득(비율 유지) 채운다. 같은 종류의 다른 코드는 우측(또는 하단) 썸네일.
8. 모든 전환은 부드러운 transition/animation.

> ⚠️ 첨부 영상은 없습니다. 위 명세 자체를 **자족적 설계 기준**으로 삼아 구현하세요. 디자인 톤은 ‘밝은 뉴모피즘 + 알록달록 색 링 + 가로 화면 가득’입니다.

## A. ⭐ 최우선 규칙 — 16:9 풀스크린 + 원본 이미지 무변형

> **이 섹션은 다른 모든 디자인 결정보다 우선합니다. AI에게 “이 규칙을 절대 어기지 말라”고 못박으세요.**

### A.1 16:9 풀스크린 레이아웃 (화면 가득)
- 앱 루트는 **뷰포트 전체**를 채운다: `width: 100dvw; height: 100dvh; overflow: hidden;`
- **16:9 가로 화면**(예: 1920×1080, 1366×768, 모니터·TV·프로젝터·태블릿 가로)을 **1순위 타깃**으로 설계한다.
- 메인·갤러리·상세 **세 화면 모두 한 화면에 들어오게**(기본은 무스크롤). 항목이 많으면 **본문 영역만** 내부 스크롤하고 헤더/프레임은 고정.
- 레이아웃은 `flex`/`grid`로 **남는 공간을 꽉 채운다.** 고정 px 폭 컨테이너(`max-w-[1180px]` 같은 좁은 박스)로 가운데만 쓰지 말고, **화면 비율에 맞춰 늘어나게** 한다.
- 두 가지 모드를 모두 지원하고, 기본은 (a):
  - **(a) 풀블리드(full-bleed) — 권장**: 앱이 실제 뷰포트를 그대로 채운다. 창이 정확히 16:9가 아니어도 콘텐츠가 자연스럽게 가득 찬다.
  - **(b) 고정 16:9 스테이지 — 옵션(키오스크/발표용)**: `aspect-ratio: 16 / 9`인 ‘무대’를 화면 가운데 두고, 뷰포트가 16:9가 아니면 위아래(또는 좌우)에 레터박스. 무대 안의 콘텐츠가 가득 차게.
- 안전영역/모바일 대응: `100dvh`(주소창 변동 대응) + `env(safe-area-inset-*)` 패딩. 세로(portrait) 기기에서는 “가로로 돌려주세요” 힌트 또는 세로 폴백 레이아웃을 둔다.

```css
/* 풀블리드 루트 */
#app-root {
  width: 100dvw;
  height: 100dvh;
  overflow: hidden;
  display: flex;
  flex-direction: column; /* 헤더(상단 고정) + 본문(flex-1로 나머지 가득) */
}
#app-header { flex: 0 0 auto; }
#app-main   { flex: 1 1 auto; min-height: 0; display: flex; } /* min-height:0 핵심: 자식 스크롤 허용 */

/* (옵션) 고정 16:9 스테이지 */
.stage-16x9 {
  width: min(100dvw, calc(100dvh * 16 / 9));
  height: min(100dvh, calc(100dvw * 9 / 16));
  margin: auto;            /* 레터박스 가운데 정렬 */
  aspect-ratio: 16 / 9;
  overflow: hidden;
}
```

### A.2 원본 코드 이미지 ‘무변형’ 표시 (가장 중요)
원본 우쿨렐레 코드 이미지를 **절대 늘이거나(stretch) 잘라내지(crop) 않는다.** **원본 비율 그대로, 전체가 다 보이게** 표시한다.

- 표시 규칙(고정):
```css
.chord-image {
  max-width: 100%;
  max-height: 100%;
  width: auto;
  height: auto;
  object-fit: contain;     /* ✅ 비율 유지 + 전체 표시 (잘림/왜곡 없음) */
  object-position: center;
  display: block;
}
.chord-image-frame {       /* 16:9 본문 안에서 이미지를 담는 영역 */
  display: flex;
  align-items: center;
  justify-content: center;
  overflow: hidden;        /* 레터박스 ‘여백’만 정리, 이미지 자체는 안 잘림 */
}
```
- **금지 목록(절대 사용 금지)**:
  - `object-fit: cover` → 이미지가 **잘림**.
  - `object-fit: fill` / `width`·`height`를 원본과 다른 비율로 동시 고정 → **왜곡(찌그러짐)**.
  - `transform: scaleX()`·`scaleY()`를 따로 줘서 비율 깨뜨리기.
  - 이미지에 `border-radius`로 콘텐츠를 잘라 운지점이 사라지게 하기. (둥근 모서리/그림자는 **프레임 배경에만**.)
- 컨테이너(프레임)는 16:9 안에서 어떤 크기든 가질 수 있지만, **이미지는 자기 비율을 유지**하고 남는 공간은 **레터박스(여백)**로 둔다. 이것이 “변형 없이 그대로” 표시하는 정석이다.
- `<img>`에 가로/세로 픽셀을 강제로 박지 않는다(또는 박더라도 위 `object-fit: contain`을 반드시 유지). 원본 intrinsic(고유) 비율을 따르게.
- **상세 화면**: 이미지가 세로 공간을 가득 채우되 비율 유지 — `height: 100%; width: auto; max-width: 100%;`. 가로가 남으면 가운데 정렬 + 좌우 여백(레터박스).
- **화질**: 확대 시 흐려짐을 막기 위해 가능하면 **SVG 원본** 또는 충분한 해상도의 **PNG/WebP**. 다이어그램은 `image-rendering: auto`, 픽셀아트 풍이면 `image-rendering: crisp-edges`.
- 이미지가 없을 때만 동적 SVG로 그린다(§7). **있으면 무조건 원본 이미지를 무변형으로** 보여준다.

> 🔒 **AI에게 못박을 문장**: *“원본 코드 이미지는 object-fit: contain으로만 표시하고, cover/fill·비율이 다른 고정 크기·일방향 scale·이미지 자체 클리핑을 절대 쓰지 마라. 화면이 16:9가 아니어도 이미지는 비율을 유지하고 남는 공간은 레터박스로 둬라.”*

## 4. 화면 구성 & 사용자 흐름 (3단계)

> 모든 화면은 **§A.1 16:9 풀스크린 셸(`AppShell`)** 안에 들어간다. 헤더는 상단에 고정, 그 아래 본문이 **남는 공간을 전부** 차지한다.

### 4.1 수준1 — 메인 (코드 종류 선택)
- 헤더: 돋보기 아이콘 + 둥근 검색창(연핑크 테두리) + 우측 `Lesson Designer`.
- 본문: **코드 종류 12개**를 **2행 × 6열** 원형 링 버튼으로 배치하되, **16:9 가로폭을 활용해 큼직하게** 채운다. 버튼 사이 간격은 `gap`으로, 전체는 본문 영역에 세로/가로 중앙 정렬.
- hover/클릭 시 부드럽게 확대(`scale`), active 상태는 더 크게 + 링 강조.

### 4.2 수준2 — 갤러리 (루트음 선택)
- 선택한 종류(예: Major)의 코드를 **카드 그리드**로 표시 (`C`, `D`, `E`, `F`, `G`, `A`, `B`).
- 상단에 현재 종류 타이틀.
- 카드 그리드는 본문 영역을 가득 채우게 `grid` 자동 열(`auto-fill/minmax`)로 펼친다. 카드 수가 적으면 가운데 정렬.
- 각 카드 = 코드명 라벨 + **미니 다이어그램(원본 이미지 무변형 또는 SVG)**.
- 카드 클릭 → 수준3로 줌인 전환.

### 4.3 수준3 — 상세 (대형 원본 다이어그램)
- 레이아웃(가로 16:9):
  - **왼쪽~중앙(약 70~75%)**: 큰 코드명 제목 + **원본 다이어그램이 비율 그대로 가득**(`object-fit: contain`, 레터박스 허용).
  - **오른쪽(약 25~30%)**: 같은 종류 다른 코드 **썸네일 rail**(세로 스크롤) → 클릭 시 즉시 전환.
  - (세로 화면 폴백) 썸네일을 **하단 가로 스크롤**로.
- 좌상단 루트 라벨(예: `|C`).
- **뒤로/홈 버튼**, **ESC 키**로 갤러리 복귀.

## 5. 디자인 명세 (16:9 레이아웃 포함)

### 5.1 전체 톤
- 배경: `#FFFFFF`~`#FBFBFB`.
- **뉴모피즘**: 위쪽 하이라이트 + 아래쪽 부드러운 그림자.
  - 예: `box-shadow: 6px 6px 14px rgba(0,0,0,.08), -6px -6px 14px rgba(255,255,255,.9);`
- 밝고 컴팩트, 넓은 여백. 단, **여백 때문에 화면이 비어 보이지 않게** 본문이 16:9를 채우도록.

### 5.5 코드 다이어그램(폴백 SVG) — 우쿨렐레 4현 세로형 ⭐
> 원본 이미지가 있으면 **이미지를 무변형으로** 쓰고, 없을 때만 아래 SVG로 그린다. 우쿨렐레는 **세로형(vertical) 다이어그램**이 표준이다.

- **현(string)**: **4개 세로선**. 왼→오 순서 **`4번현 G` → `3번현 C` → `2번현 E` → `1번현 A`**. (1번현=가장 얇은 줄=맨 오른쪽)
- **프렛(fret)**: **가로선**. 위에서 아래로 1·2·3·4프렛. 맨 위는 **너트(두꺼운 가로 막대)**.
- **운지 표시**: 누르는 칸 중앙에 **채워진 원(dot)** + 그 안에 finger 번호(흰 글자). 옅은 캡슐 텍스처(선택).
- **개방현/뮤트**: 너트 위 각 현에 개방현 `o`, 안 치는 줄 `x`.
- **루트 라벨**: 좌상단(또는 상단 중앙)에 코드 루트(예: `C`).
- **baseFret**: 1프렛부터가 아니면 너트 막대를 빼고 우측에 `5fr` 같은 시작 프렛 라벨.
- **줄 라벨**: 하단(또는 상단)에 `G C E A`.

**SVG 좌표 가이드(예시, viewBox `0 0 240 300`)**
```
- 4개 세로 현 x ≈ 50, 100, 150, 200  (왼=G ... 오=A), y 60→260
- 너트: y≈60 두꺼운 가로 막대, 프렛선 y≈110/160/210/260 (간격 50)
- 운지 dot: 해당 현 x, (fret n-1 ~ n) 중앙 y, r≈14, 안에 finger 번호
- 개방현/뮤트: y≈40 (너트 위), 각 현 x
- 코드명: 상단 y≈24 가운데, 줄 라벨 G C E A: y≈285
```

### 5.6 애니메이션
1. 원형 버튼 hover: `scale 1 → 1.06`, 그림자 강화
2. 원형 버튼 active: `scale 1.1`, 링 두께/글로우 강화
3. 종류 선택 → 갤러리 등장: `opacity 0→1`, `translateY 12px→0`
4. 코드 카드 hover: `scale 1 → 1.04`
5. 상세 진입: 큰 이미지 `opacity 0→1`, `scale .98→1` (※ 단방향 scale로 비율을 깨지 않게, transform-origin center + 균등 scale만)
6. 우측 썸네일: staggered 등장
- 전부 `transition: 0.2~0.3s ease`.

## 6. 데이터 구조 (우쿨렐레 4현)

운지 정보를 데이터로 두면 원본 이미지·동적 SVG 어느 쪽이든 같은 데이터로 처리할 수 있습니다.

### 6.1 타입 정의 (`src/data/chordTypes.ts`)
```ts
export type ChordQualityId =
  | 'major' | 'dominant7' | 'minor' | 'minor7' | 'sus4' | 'major7'
  | 'six' | 'sevenSus4' | 'add2' | 'm7b5' | 'diminish' | 'augment';

// ⭐ 우쿨렐레 현 컨벤션 (한 번에 고정)
// string: 1 = A(가장 얇은 줄, 다이어그램 맨 오른쪽)
//         2 = E, 3 = C, 4 = G(다이어그램 맨 왼쪽)
//   ※ 표준 high-G(재진입) 튜닝 기준. 음높이가 아니라 "현 번호"로 통일한다.
// fret:   0 = 개방현, N = N번 프렛
export type FingerPosition = {
  string: 1 | 2 | 3 | 4;
  fret: number;
  finger?: 1 | 2 | 3 | 4;            // 검지1 ~ 새끼4
  open?: boolean;                    // 개방현(o)
  muted?: boolean;                   // 안 치는 줄(x)
};

export type ChordShape = {
  id: string;                        // 'c_major'
  root: string;                      // 'C'
  quality: ChordQualityId;
  title: string;                     // 'C Major'
  positions: FingerPosition[];       // 4개 현에 대한 운지
  baseFret?: number;                 // 다이어그램 시작 프렛(기본 1)
  image?: string;                    // ⭐ (1순위) 원본 이미지 경로. 있으면 이미지 우선, 없으면 SVG
  difficulty?: 'beginner' | 'intermediate' | 'advanced';
  tags?: string[];
};
```

> ⚠️ **현 번호 컨벤션을 한 가지로 고정**하라고 AI에게 강조하세요. `string` 값을 명시하는 객체형이라 좌우 뒤집힘 실수를 줄여줍니다.

```ts
      { string: 3, fret: 0, open: true },   // C
      { string: 2, fret: 1, finger: 1 },    // E 1프렛
      { string: 1, fret: 0, open: true },   // A
    ],
  },
  {
    id: 'g_major', root: 'G', quality: 'major', title: 'G Major',
    baseFret: 1, difficulty: 'beginner', tags: ['open chord'],
    image: '/chords/major/g.svg',
    positions: [
      { string: 4, fret: 0, open: true },   // G
      { string: 3, fret: 2, finger: 1 },    // C 2프렛
      { string: 2, fret: 3, finger: 3 },    // E 3프렛
      { string: 1, fret: 2, finger: 2 },    // A 2프렛
    ],
  },
];
```

## 7. ⭐ 우쿨렐레 코드 이미지 저장·표시 방법 (원본 무변형이 1순위)

> 사용자가 가장 중요하게 요청한 부분. **이번 프로젝트의 기본 = 원본 이미지(무변형 표시) 1순위 + 동적 SVG 폴백.**

### 7.0 방법 비교표 & 선택 가이드

| 방법 | 이미지를 어디에? | 백엔드 | 추가 방법 | 적합한 경우 |
|------|------------------|--------|-----------|-------------|
| **A. 원본 정적 이미지** ⭐1순위(이번 요청) | `public/chords/...` | ❌ | 파일 넣고 `chords.ts`/manifest에 경로 추가 후 재배포 | **이미 만든 원본 다이어그램을 그대로** 보여주고 싶을 때 |
| **B. 동적 SVG 생성** (폴백) | 저장 안 함(데이터로 그림) | ❌ | `chords.ts`에 운지 데이터만 추가 | 이미지가 없는 코드 자동 표시·무한 확대 |
| **C. IndexedDB 업로드** | 사용자 브라우저 내부 | ❌ | 사이트 업로드 UI | 재배포 없이 내 브라우저에서 추가/보관 |
| **D. 클라우드(Supabase 등)** | 클라우드 버킷 | ✅ | 관리자 업로드 → 전체 반영 | 여러 사용자 공유·관리자 기능 |

> 🎯 **이번 프로젝트 기본**: **A(원본 이미지) + B(폴백)**. 원본 이미지가 있으면 **무조건 무변형으로** 보여주고, 없는 코드만 SVG로.

### 7.1 [1순위] 원본 정적 이미지 — 무변형 표시
원본 우쿨렐레 코드 다이어그램 이미지를 `public/chords/...`에 넣고, `shape.image`로 경로를 연결한다. 표시는 **§A.2 규칙(`object-fit: contain`)**을 그대로 따른다.

- **경로 규칙(고정)**: `public/chords/{quality}/{root}.{ext}` (ext = `svg` | `png` | `webp`)
  ```txt
  C Major   → /chords/major/c.svg
  C# Major  → /chords/major/c-sharp.svg   (# 은 -sharp 로 치환)
  Db Major  → /chords/major/d-flat.svg
  A minor7  → /chords/minor7/a.svg
  B m7(b5)  → /chords/m7b5/b.svg
  ```
- **`ChordImage.tsx`** (무변형 표시 컴포넌트):
  ```tsx
  type Props = { src: string; alt: string; size?: 'thumb' | 'large' };

  export function ChordImage({ src, alt, size = 'thumb' }: Props) {
    return (
      <div className="chord-image-frame w-full h-full flex items-center justify-center overflow-hidden">
        <img
          src={src}
          alt={alt}
          loading="lazy"
          // ⭐ 무변형 핵심: contain + max 100% + width/height auto
          className="max-w-full max-h-full w-auto h-auto object-contain"
          onError={(e) => { e.currentTarget.src = '/chords/placeholders/chord-placeholder.svg'; }}
        />
      </div>
    );
  }
  ```
- **manifest.json** (이미지 수십 개 이상일 때):
  ```json
  {
    "version": 1,
    "basePath": "/chords",
    "items": [
      { "id": "c_major", "root": "C", "quality": "major", "title": "C Major", "imageSrc": "/chords/major/c.svg" },
      { "id": "f_major", "root": "F", "quality": "major", "title": "F Major", "imageSrc": "/chords/major/f.svg" }
    ]
  }
  ```
> 📌 포맷은 **SVG**(무한 확대로 가장 선명) 또는 **WebP/PNG**(고해상도) 권장. **크기를 강제로 통일하지 말고** 원본 비율을 그대로 두면, 표시 단계에서 `contain`이 알아서 비율을 지킨다.

### 7.2 [폴백] 데이터 기반 동적 SVG (4현 세로형)
운지 데이터(`positions`)를 받아 `Fretboard`가 SVG를 직접 그린다. **이미지가 없는 코드에만** 쓴다.

`Fretboard.tsx` 스켈레톤:
```tsx
type Props = { shape: ChordShape; size?: 'thumb' | 'large' };

export function Fretboard({ shape, size = 'thumb' }: Props) {
  // 우쿨렐레 4현 세로 다이어그램 (§5.5):
  //  - 4개 세로선(현): 왼=4번현 G ... 오=1번현 A
  //  - 가로선(프렛) + 맨 위 너트(두꺼운 가로 막대)
  //  - positions 순회: fret>0 → 해당 (현 x, 칸 중앙 y)에 채운 원 + finger 번호
  //                    open → 너트 위 'o', muted → 'x'
  //  - 상단 코드명, 하단 'G C E A' 라벨, baseFret>1이면 'Nfr'
  //  - 외부 이미지/base64 없이 순수 SVG element만
  return (/* <svg viewBox="0 0 240 300"> ... </svg> */);
}
```

### 7.3 표시 우선순위 결정 (`ChordDiagram.tsx`)
```tsx
// ① IndexedDB 업로드 이미지 → ② shape.image(정적/클라우드 원본) → ③ 동적 SVG
export function ChordDiagram({ shape, size }: { shape: ChordShape; size?: 'thumb'|'large' }) {
  const uploadedUrl = useUploadedChordImageUrl(shape.id);  // 없으면 null
  const src = uploadedUrl ?? shape.image ?? null;
  if (src) return <ChordImage src={src} alt={`${shape.title} 우쿨렐레 코드`} size={size} />;
  return <Fretboard shape={shape} size={size} />;
}
```

### 7.4 [선택] IndexedDB 업로드 — 재배포 없이 브라우저 저장
이미지는 바이너리(Blob)라 `localStorage`보다 **IndexedDB**가 적합. `idb` 패키지 사용.

```bash
npm install idb
```
`src/lib/chordImageStorage.ts`:
```ts
import { openDB } from 'idb';

export type UploadedChordImage = {
  id: string; chordId: string; root: string; quality: string; title: string;
  fileName: string; mimeType: string; blob: Blob; createdAt: string; updatedAt: string;
};

const DB_NAME = 'ukulele-chord-viewer';
const STORE = 'chord-images';

async function db() {
  return openDB(DB_NAME, 1, {
    upgrade(d) { if (!d.objectStoreNames.contains(STORE)) d.createObjectStore(STORE, { keyPath: 'id' }); },
  });
}
export async function saveUploadedChordImage(item: UploadedChordImage) { (await db()).put(STORE, item); }
export async function getUploadedChordImageUrl(id: string) {
  const item = await (await db()).get(STORE, id) as UploadedChordImage | undefined;
  return item ? URL.createObjectURL(item.blob) : null;   // <img src>에 사용 → 역시 object-fit:contain로 표시
}
```
업로드 UI 규칙: 허용 `svg/png/webp`, **2MB 이하**, 미리보기, 저장 후 즉시 반영, 새로고침 후 재로드, 삭제 지원.
**장점**: 백엔드 0, 새로고침해도 유지. **단점**: 그 브라우저/기기에서만 보임(공유 X).

### 7.5 [선택] Supabase Storage — 클라우드 공유·관리자 업로드
```ts
import { createClient } from '@supabase/supabase-js';
const supabase = createClient(URL, ANON_KEY);

await supabase.storage.from('chord-images').upload('major/c.svg', file, { upsert: true });
const { data } = supabase.storage.from('chord-images').getPublicUrl('major/c.svg');
const imageUrl = data.publicUrl;   // shape.image 에 그대로 연결 → 무변형 표시
```
앱 로딩 흐름: ① 정적 기본 데이터 로드 → ② Supabase에서 추가 메타데이터 fetch → ③ 같은 `chordId`면 클라우드 이미지가 정적 이미지를 **override**.

### 7.6 ⚠️ 반드시 알아야 할 현실적 한계
브라우저에서 이미지를 업로드해도 **배포된 정적 사이트의 `public/chords` 폴더에 파일이 써지지 않습니다.** 정적 사이트는 배포 후 서버 파일시스템을 브라우저가 직접 수정할 수 없습니다. 그래서 저장은 목적별로 나눕니다.
1. **개발자 기본 이미지**: `public/chords`에 직접 넣고 Git 관리.
2. **사용자 개인 추가**: IndexedDB(그 브라우저에서만).
3. **전체 공유**: Supabase/S3/R2 등 object storage + DB.

> README와 코드 주석에 다음 문장을 **반드시** 포함하세요:
> *“정적 배포된 웹사이트의 public 폴더는 브라우저에서 직접 수정할 수 없다. 기본 이미지는 개발자가 프로젝트에 포함해 배포하고, 사용자 업로드 이미지는 IndexedDB 또는 서버 스토리지에 저장해야 한다.”*

```ts
const [searchTerm, setSearchTerm] = useState('');
```

## 9. 접근성 & 반응형 (16:9 우선)

**접근성**: 클릭 요소는 `button` 또는 키보드 핸들러 / 모든 이미지 `alt`(예: `"C Major 우쿨렐레 코드 다이어그램"`) / 검색창 `aria-label="Search ukulele chords"` / 선택된 종류 `aria-pressed` / 색만으로 상태 구분 금지(ring·scale·텍스트 병행) / **ESC로 상세 닫기** / `focus-visible` ring.

**반응형 (16:9 풀스크린 기준)**
| 구분 | 셸 | 원형 버튼 | 갤러리 grid | 상세 |
|------|----|-----------|-------------|------|
| **가로 데스크톱/TV (16:9 기본)** | `100dvw×100dvh` 풀블리드 | `clamp(96px,12vh,140px)` | `auto-fill minmax(180px,1fr)` 본문 가득 | 좌측 큰 이미지(contain) + 우측 320px 썸네일 rail |
| 태블릿 가로 | 동일 | `clamp(88px,12vh,120px)` | 3~4열 | 좌측 이미지 + 우측 좁은 rail |
| 모바일/세로 | `100dvh`, 세로 폴백 | `86~96px` wrap | 2열 | 큰 이미지(contain) + **하단 가로 스크롤 썸네일** |

> 세로 화면에서는 “가로로 돌려 보면 더 좋다”는 힌트를 보여주거나, 위 세로 폴백 레이아웃으로 자연스럽게 전환한다. **어느 경우든 이미지는 비율 유지(contain).**

## 13. 운영 배포 주의사항
- 디자인 자산(로고/이미지)은 **직접 제작**하거나 라이선스가 명확한 것만 사용한다.
- 외부 코드 다이어그램 이미지를 무단 복사하지 말고, **직접 만든 원본**이나 라이선스가 명확한 자산을 쓴다. (직접 만든 원본 이미지 + 동적 SVG가 저작권상 안전)
- 정적 배포만 하면 업로드 이미지는 다른 사용자에게 공유되지 않는다(공유하려면 서버/스토리지 필요).

- 코드 수백 개까지는 클라이언트 필터링으로 충분, 수천 개면 인덱싱/서버 검색 고려.

## 부록 A. 표준 우쿨렐레 오픈 코드 운지표

AI가 `chords.ts`를 채울 기준 표입니다. 표기는 **`4번현 → 1번현` = `G C E A`** 순서이며,
§6.1 `positions`(객체형)로 변환할 때 **string 값을 명시**하므로 좌우 뒤집힘 걱정이 없습니다.
(`0`=개방현, 숫자=프렛, `x`=뮤트. 표준 high-G 튜닝 기준)

| 코드 | 운지 (G C E A) | 운지 손가락(예) |
|------|----------------|-----------------|
| C Major | `0 0 0 3` | A현 3프렛(3) — 가장 쉬움 |
| C7 | `0 0 0 1` | A현 1프렛(1) |
| C Major7 | `0 0 0 2` | A현 2프렛(2) |
| C minor | `0 3 3 3` | C·E·A현 3프렛 |
| D Major | `2 2 2 0` | G·C·E현 2프렛(1·2·3) |
| D7 | `2 2 2 3` | + A현 3프렛 |
| D minor | `2 2 1 0` | G2·C2·E1 |
| E Major | `4 4 4 2` | (쉬운 대안 `1 4 0 2`) |
| E7 | `1 2 0 2` | G1·C2·A2 |
| E minor | `0 4 3 2` | C4·E3·A2 |
| F Major | `2 0 1 0` | G현 2프렛(2), E현 1프렛(1) |
| F7 | `2 3 1 0` | G2·C3·E1 |
| F minor | `1 0 1 3` | G1·E1·A3 |
| G Major | `0 2 3 2` | C2·E3·A2 |
| G7 | `0 2 1 2` | C2·E1·A2 |
| G minor | `0 2 3 1` | C2·E3·A1 |
| A Major | `2 1 0 0` | G현 2프렛(2), C현 1프렛(1) |
| A7 | `0 1 0 0` | C현 1프렛(1) |
| A minor | `2 0 0 0` | G현 2프렛(2) — 한 손가락 |
| A minor7 | `0 0 0 0` | 전부 개방현 |
| B Major | `4 3 2 2` | (살짝 바레 느낌) |
| B7 | `2 3 2 2` | C현만 3프렛 |
| Bb Major | `3 2 1 1` | E·A현 1프렛 바레 |
| B minor | `4 2 2 2` | G현 4프렛 |

> AI에게: “위 표를 §6.1 `ChordShape.positions` 형식으로 변환해 `chords.ts`에 넣어라. (string 1=A 얇은 줄, 4=G 굵은 줄 / fret 0=open / 표의 x=muted) 각 코드의 `image`는 `/chords/{quality}/{root}.svg`로 연결하고, 이미지가 없으면 동적 SVG로 폴백되게 하라. 나머지 종류(sus4, 6, add2, m7(b5), diminish, Augment 등)도 표준 오픈 운지로 채워라.”
